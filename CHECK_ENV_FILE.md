# 🔍 .env.localファイル確認ガイド

## ❌ 現在の状態

`.env.local`ファイルが見つかりませんでした。

---

## 🔍 確認方法

### 方法1: VS Codeで確認

1. **VS Codeで`blog-writer-app`フォルダを開く**

2. **エクスプローラーパネルで確認**
   - 左側のファイル一覧を確認
   - `.env.local`というファイルがあるか確認

3. **隠しファイルを表示**
   - VS Codeの設定で「隠しファイルを表示」が有効になっているか確認
   - ファイル名が`.env.local`で始まるか確認（ドットで始まる）

### 方法2: Windowsエクスプローラーで確認

1. **`blog-writer-app`フォルダを開く**

2. **表示設定を変更**
   - 「表示」タブをクリック
   - 「隠しファイル」にチェックを入れる

3. **`.env.local`ファイルを探す**
   - ファイル名が`.env.local`であることを確認
   - ファイル名に余分な文字（`.txt`など）がついていないか確認

### 方法3: よくある間違い

以下のようなファイル名になっていないか確認してください：

❌ **間違ったファイル名**:
- `env.local`（ドットがない）
- `.env.local.txt`（`.txt`がついている）
- `.env`（`.local`がない）
- `env.local.txt`（ドットがない + `.txt`がついている）

✅ **正しいファイル名**:
- `.env.local`（ドットで始まり、`.local`で終わる）

## 📝 正しい作成方法

### VS Codeで作成する場合

1. **VS Codeで`blog-writer-app`フォルダを開く**

2. **新しいファイルを作成**
   - エクスプローラーパネルで右クリック
   - 「新しいファイル」を選択
   - ファイル名を **`.env.local`** と入力（ドットで始まる）

3. **内容を記述**
   ```env
   # Supabase settings
   NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
   ```

4. **保存**
   - `Ctrl + S` で保存
   - 文字コードはUTF-8で保存

### Windowsエクスプローラーで作成する場合

1. **`blog-writer-app`フォルダを開く**

2. **新規テキストファイルを作成**
   - 右クリック → 「新規」→ 「テキスト ドキュメント」

3. **ファイル名を変更**
   - ファイル名を `.env.local` に変更
   - 警告が表示されたら「はい」をクリック
   - **重要**: `.txt`拡張子を削除する必要があります

4. **内容を記述**
   - メモ帳で開く
   - 上記の内容を記述
   - 保存

## ✅ 確認チェックリスト

ファイルを作成したら、以下を確認してください：

- [ ] ファイル名が`.env.local`である（ドットで始まる）
- [ ] ファイルが`blog-writer-app`フォルダにある
- [ ] ファイルに`NEXT_PUBLIC_SUPABASE_URL`が設定されている

## 🚨 よくある問題

### 問題1: ファイルが見つからない

**原因**: ファイル名が間違っている、または別の場所に保存されている

**解決方法**:
1. ファイル名を確認（`.env.local`であること）
2. 保存場所を確認（`blog-writer-app`フォルダ内であること）
3. 隠しファイルを表示する設定を確認

### 問題2: ファイル名に`.txt`がついている

**原因**: Windowsが自動的に`.txt`拡張子を追加している

**解決方法**:
1. ファイル名を変更
2. 「表示」→「ファイル名拡張子」にチェック
3. ファイル名から`.txt`を削除

### 問題3: 環境変数が読み込まれない

**原因**: サーバーを再起動していない

**解決方法**:
1. サーバーを停止（`Ctrl + C`）
2. サーバーを再起動（`npm run dev`）
3. ブラウザをリロード
