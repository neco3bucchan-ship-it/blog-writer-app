# 📖 VS Code（Visual Studio Code）使い方ガイド

## 🎯 このガイドの目的

非エンジニアの方でも、VS Codeを使って`.env.local`ファイルを作成・編集できるように、わかりやすく説明します。

---

## 📥 Step 1: VS Codeをインストールする（まだインストールしていない場合）

### 1.1 VS Codeをダウンロード

1. **ブラウザを開く**
   - Google Chrome、Microsoft Edge、Firefoxなど、お使いのブラウザを開く

2. **VS Codeの公式サイトにアクセス**
   - アドレスバーに以下のURLを入力してEnterキーを押す：
     ```
     https://code.visualstudio.com/
     ```

3. **ダウンロードボタンをクリック**
   - 画面に「Download for Windows」という大きなボタンがあるので、クリック
   - 自動的にダウンロードが始まります

### 1.2 VS Codeをインストール

1. **ダウンロードしたファイルを開く**
   - 通常は「ダウンロード」フォルダに保存されます
   - ファイル名は `VSCodeUserSetup-x64-xxxxx.exe` のような名前です

2. **インストールウィザードを実行**
   - ファイルをダブルクリック
   - 「同意する」を選択して「次へ」をクリック
   - インストール場所はそのままで「次へ」
   - 「次へ」を何度かクリック
   - 「インストール」をクリック
   - インストールが完了したら「完了」をクリック

## 🚀 Step 2: VS Codeを開く

### 方法1: スタートメニューから開く（最も簡単）

1. **Windowsのスタートボタンをクリック**
   - 画面左下のWindowsアイコンをクリック

2. **「Visual Studio Code」を検索**
   - スタートメニューが開いたら、キーボードで「Visual Studio Code」と入力
   - または、アプリ一覧から「Visual Studio Code」を探す

3. **VS Codeをクリック**
   - 「Visual Studio Code」というアプリをクリック
   - VS Codeが起動します

### 方法2: デスクトップのショートカットから開く

1. **デスクトップを確認**
   - デスクトップに「Visual Studio Code」というアイコンがあるか確認

2. **アイコンをダブルクリック**
   - アイコンをダブルクリックすると、VS Codeが起動します

### 方法3: フォルダから直接開く

1. **Windowsエクスプローラーを開く**
   - 画面下部のタスクバーにあるフォルダアイコンをクリック
   - または、`Windowsキー + E` を押す

2. **`blog-writer-app`フォルダを開く**
   - 以下のパスに移動：
     ```
     C:\Users\t-ich\Documents\いまにゅClaud AI Cording202507\cord4biz AI cording_20250821から\Webアプリ開発\blog-generator\blog-writer-app
     ```

3. **フォルダ内で右クリック**
   - フォルダ内の空白部分で右クリック
   - 「Codeで開く」または「Open with Code」を選択
   - VS Codeが起動して、そのフォルダが開きます

## 📂 Step 3: プロジェクトフォルダを開く

### 3.1 VS Codeでフォルダを開く

1. **VS Codeが起動したら**
   - 画面左上の「ファイル」メニューをクリック
   - 「フォルダーを開く」を選択

2. **フォルダを選択**
   - ファイル選択ダイアログが開きます
   - 以下のパスに移動：
     ```
     C:\Users\t-ich\Documents\いまにゅClaud AI Cording202507\cord4biz AI cording_20250821から\Webアプリ開発\blog-generator\blog-writer-app
     ```
   - `blog-writer-app`フォルダを選択
   - 「フォルダーの選択」ボタンをクリック

3. **フォルダが開いたら**
   - 画面左側にファイル一覧が表示されます
   - これで準備完了です！

## 📝 Step 4: .env.localファイルを作成する

### 4.1 新しいファイルを作成

1. **左側のファイル一覧（エクスプローラー）を確認**
   - 画面左側にファイルやフォルダの一覧が表示されています
   - これが「エクスプローラー」パネルです

2. **`blog-writer-app`フォルダを確認**
   - エクスプローラーの一番上に`blog-writer-app`というフォルダ名が表示されています

3. **右クリックして新しいファイルを作成**
   - `blog-writer-app`フォルダ名の上で右クリック
   - メニューが表示されるので、「新しいファイル」をクリック
   - または、`blog-writer-app`フォルダ内の空白部分で右クリック → 「新しいファイル」

4. **ファイル名を入力**
   - 新しいファイルが作成され、ファイル名を入力できる状態になります
   - 以下の名前を入力：
     ```
     .env.local
     ```
   - **重要**: 最初にドット（`.`）を入力してください
   - Enterキーを押すと、ファイルが作成されます

### 4.2 ファイルに内容を記述

1. **ファイルを開く**
   - 作成した`.env.local`ファイルをクリック
   - 画面中央のエディタ部分にファイルが開きます

2. **以下の内容をコピー＆ペースト**
   ```env
   # Supabase settings
   NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
   ```

3. **実際の値を設定**
   - `https://your-project-id.supabase.co` の部分を、実際のSupabase URLに置き換える
   - `your-anon-key-here` の部分を、実際のSupabase APIキーに置き換える

### 4.3 ファイルを保存

1. **保存する**
   - `Ctrl + S` キーを押す
   - または、画面左上の「ファイル」→「保存」をクリック

2. **保存確認**
   - 画面右上のタブに、ファイル名の横に白い丸（未保存の印）が消えれば保存完了です

## 🔍 Step 5: ファイルが見つからない場合

### 5.1 隠しファイルを表示する

1. **VS Codeの設定を確認**
   - 画面左下の歯車アイコン⚙️をクリック
   - 「設定」を選択

2. **検索ボックスで検索**
   - 設定画面の上部に検索ボックスがあるので、以下を入力：
     ```
     files.exclude
     ```

3. **設定を確認**
   - `.env*` という項目があるか確認
   - チェックが入っている場合は、外す

### 5.2 ファイル名を確認

- ファイル名が正確に `.env.local` になっているか確認
- 余分な文字（`.txt`など）がついていないか確認

## 💡 便利な機能

### ファイルを検索する

- `Ctrl + P` キーを押すと、ファイル名で検索できます
- `.env.local` と入力すると、ファイルが見つかります

### ターミナルを開く

- `Ctrl + Shift + `` （バッククォート）キーを押すと、ターミナルが開きます
- ここで `npm run dev` などのコマンドを実行できます

### ファイルを閉じる

- タブの右上の×ボタンをクリック
- または、`Ctrl + W` キーを押す

## ❓ よくある質問

### Q1: VS Codeが起動しない

**A**: 
- インストールが完了しているか確認
- 再起動してみる
- 管理者権限で実行してみる

### Q2: フォルダが見つからない

**A**: 
- Windowsエクスプローラーで、フォルダの場所を確認
- パスをコピーして、VS Codeの「フォルダーを開く」で貼り付ける

### Q3: ファイルが保存できない

**A**: 
- ファイルが読み取り専用になっていないか確認
- 管理者権限でVS Codeを開いてみる

### Q4: ファイル名にドットが入力できない

**A**: 
- 日本語入力モードをオフにする（半角英数モード）
- `Shift + 2` でドット（`.`）を入力
