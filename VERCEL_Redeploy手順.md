# 🔄 Vercel 再デプロイ（Redeploy）手順

## 現在の状況

「Deployments」タブで、デプロイメントが「Ready」状態で表示されています。
ID: `9LdxnrKJA`、環境: Production、ステータス: Ready

## ✅ 再デプロイの手順

### Step 1: デプロイメントの3点リーダーメニューを開く

1. **デプロイメントリストを確認**
   - 「Deployments」タブで、再デプロイしたいデプロイメントを探します
   - 現在表示されているデプロイメント（ID: `9LdxnrKJA`）の右端を確認してください

2. **3点リーダーメニューをクリック**
   - デプロイメントエントリの右端に「⋯」（3点リーダー）アイコンがあります
   - このアイコンをクリックします

### Step 2: 「Redeploy」を選択

3点リーダーメニューを開くと、以下のようなオプションが表示されます：

- **Redeploy** ← これを選択
- **View Build Logs**
- **Download**
- **Promote to Production**（該当する場合）
- **Delete**（該当する場合）

「Redeploy」をクリックします。

### Step 3: 確認と実行

1. **確認ダイアログが表示される場合**
   - 「Redeploy this deployment?」などの確認メッセージが表示される場合があります
   - 「Confirm」または「Redeploy」ボタンをクリックします

2. **再デプロイの開始**
   - 再デプロイが開始されると、デプロイメントのステータスが「Building」に変わります
   - ビルドログを確認できます

### Step 4: デプロイ完了を待つ

1. **ステータスの確認**
   - デプロイメントリストで、新しいデプロイメントが作成されます
   - または、既存のデプロイメントのステータスが更新されます
   - ステータスが「Ready」になるまで待ちます（通常2-5分）

2. **ビルドログの確認（オプション）**
   - デプロイメントをクリックすると、ビルドログを確認できます
   - エラーがないか確認してください

## 📸 視覚的な手順

```
1. Deploymentsタブを開く
   ↓
2. デプロイメントエントリの右端の「⋯」をクリック
   ↓
3. メニューから「Redeploy」を選択
   ↓
4. 確認ダイアログで「Confirm」をクリック
   ↓
5. ビルドが開始され、ステータスが「Building」に変わる
   ↓
6. ステータスが「Ready」になるまで待つ
```

## 🔍 代替方法：デプロイメントをクリックして再デプロイ

1. **デプロイメントをクリック**
   - デプロイメントエントリ自体をクリックします
   - デプロイメントの詳細ページが開きます

2. **詳細ページで再デプロイ**
   - デプロイメント詳細ページの右上に「Redeploy」ボタンがある場合があります
   - または、3点リーダーメニューから「Redeploy」を選択できます

## ⚠️ 注意事項

- **Root Directoryの設定を変更した場合**
  - Root Directoryを `blog-writer-app` に設定した後、必ず再デプロイを実行してください
  - 設定を変更しただけでは、既存のデプロイメントには反映されません

- **環境変数を変更した場合**
  - 環境変数を追加・変更した後も、再デプロイが必要です

- **ビルド時間**
  - 再デプロイには通常2-5分かかります
   - ビルドログで進行状況を確認できます

## 📝 確認チェックリスト

再デプロイが完了したら、以下を確認：

- [ ] デプロイメントのステータスが「Ready」になっている
- [ ] ビルドログにエラーがない
- [ ] URLにアクセスしてランディングページが表示される
- [ ] 404エラーが解消されている
